KEY FEATURES EXPLAINED
================================================================================

ORCHESTRATOR (Level 0):
  - Main agent that routes all requests
  - Never executes tasks directly
  - Classifies intent and delegates to specialists
  - Detects brand and auth requirements

BRAND LAYER (NEW):
  - Injects persona across all layers
  - PB: Sophisticated, editorial, aspirational
  - PBK: Reassuring, safety-first, certification-aware
  - PBT: Direct, trend-aware, peer-adjacent

SPECIALISTS (Level 1):
  - Discovery: Pre-purchase exploration (5 sub-agents)
  - FAQ: Knowledge Q&A and policy
  - Checkout: Transactional flows
  - Order Tracking: Post-purchase anxiety (auth-gated)
  - Post-Order: Returns, refunds, warranty claims

DATA LAYER:
  - Product catalog with specs and certifications
  - Order API (auth-gated)
  - Salesforce Data Cloud for personalization
  - The Key loyalty program integration
  - Brand context and policy rules

UI RENDER LAYER (NEW):
  - Separates content generation from presentation
  - Brand-specific display rules
  - PB: Large imagery, mood boards
  - PBK: Safety badges, cert tables
  - PBT: Compact cards, "shop the vibe"

HUMAN ESCALATION:
  - Feature, not fallback
  - Pre-loads context before handoff
  - Design consultation, warranty claims, auth failures
  - Shopper feels handed off, not abandoned

================================================================================
CUSTOMIZATION
================================================================================

To modify utterances:
  → Edit UTTERANCES array in app.js
  → Add new example with classification rules

To change intent patterns:
  → Edit classifyUtterance() function in app.js
  → Modify regex patterns to detect new intents

To customize styles:
  → Edit CSS variables at top of index.html
  → Adjust colors, fonts, spacing

To add new layers:
  → Add layer to LAYERS array
  → Create section in buildLayerSections()
